## 二、入口文件

### `app.py`

作用：创建 Flask 应用并启动。

核心逻辑：

```python
from genealogy_app import create_app

app = create_app()
```

讲解方式：

`app.py` 是整个 Web 应用的入口。它调用 `genealogy_app/__init__.py` 中的 `create_app()` 创建 Flask 应用。

## 三、后端应用目录 `genealogy_app/`

### 1. `genealogy_app/__init__.py`

作用：创建和配置 Flask 应用。

主要负责：

- 设置 `SECRET_KEY`；
- 设置数据库连接 `DATABASE_URL`；
- 初始化 SQLAlchemy；
- 注册路由蓝图；
- 自动创建数据库表。

讲解方式：

这个文件相当于应用工厂。程序启动时会读取环境变量，如果没有配置 PostgreSQL，就默认使用 SQLite。

关键配置：

```python
app.config["SQLALCHEMY_DATABASE_URI"] = os.getenv(
    "DATABASE_URL",
    "sqlite:///genealogy_demo.db",
)
```

说明：

如果设置了 `DATABASE_URL`，系统连接 PostgreSQL；否则使用本地 SQLite。

---

### 2. `genealogy_app/models.py`

作用：定义数据库表对应的 Python 类。

主要模型：

| 类 | 对应表 | 含义 |
|---|---|---|
| `User` | `users` | 用户 |
| `Genealogy` | `genealogies` | 族谱 |
| `GenealogyCollaborator` | `genealogy_collaborators` | 协作者关系 |
| `Member` | `members` | 族谱成员 |
| `ParentChildRelation` | `parent_child_relations` | 亲子关系 |
| `Marriage` | `marriages` | 婚姻关系 |

讲解方式：

这个文件是 ORM 映射。每个类对应数据库中的一张表，类里的字段对应表字段。代码通过操作这些类完成增删改查。

---

### 3. `genealogy_app/routes.py`

作用：处理页面请求和表单提交。

这是项目中最主要的控制层代码。

主要功能：

| 路由/函数 | 功能 |
|---|---|
| `register()` | 用户注册 |
| `login()` | 用户登录 |
| `dashboard()` | 登录后的控制台 |
| `create_genealogy()` | 创建族谱 |
| `genealogy_detail()` | 族谱详情页 |
| `create_member()` | 新增成员 |
| `edit_member()` | 编辑成员 |
| `delete_member()` | 删除成员 |
| `create_relation()` | 维护亲子和婚姻关系 |
| `search_members()` | 姓名模糊查询 |
| `tree_preview()` | 树状族谱展示 |
| `ancestor_query()` | 祖先查询 |
| `kinship_query()` | 亲缘路径查询 |
| `member_options()` | 成员搜索下拉接口 |

讲解方式：

这个文件负责“用户点页面按钮后发生什么”。例如用户提交新增成员表单，`create_member()` 会读取表单内容，创建 `Member` 对象，再写入数据库。

---

### 4. `genealogy_app/services.py`

作用：放比较复杂的业务查询。

主要函数：

| 函数 | 功能 |
|---|---|
| `user_can_access_genealogy()` | 判断用户是否能访问某个族谱 |
| `dashboard_stats()` | 统计某个族谱总人数、男女比例 |
| `query_subtree()` | 查询某个成员下方的分支树 |
| `query_ancestors()` | 递归查询某成员祖先 |
| `find_kinship_path()` | 查询两名成员之间的亲缘路径 |

讲解方式：

这个文件是业务查询层。普通页面跳转放在 `routes.py`，复杂查询放在 `services.py`，这样结构更清晰。

## 四、核心功能代码怎么对应

### 1. 登录和权限

相关文件：

```text
genealogy_app/routes.py
genealogy_app/services.py
```

实现方式：

- 登录成功后把用户 ID 存入 `session`；
- 访问族谱前调用 `ensure_access()`；
- `ensure_access()` 内部调用 `user_can_access_genealogy()`；
- 如果用户既不是创建者，也不是协作者，就不能访问。

汇报时可以说：

系统通过 Flask Session 保存登录状态，通过族谱创建者和协作者表共同判断访问权限。

---

### 2. 成员新增和关系挂接

相关文件：

```text
genealogy_app/routes.py
templates/member_form.html
templates/_member_picker.html
```

实现方式：

- 页面中输入姓名搜索父亲、母亲、配偶；
- 前端选择后提交隐藏的成员 ID；
- 后端创建新成员；
- 再根据表单中的父亲、母亲、配偶字段创建关系。

汇报时可以说：

新增成员时不只录入成员基本信息，还可以直接建立亲子或婚姻关系，提高操作效率。

---

### 3. 成员搜索下拉

相关文件：

```text
templates/_member_picker.html
templates/base.html
genealogy_app/routes.py
```

实现方式：

- `_member_picker.html` 定义通用成员选择控件；
- `base.html` 中的 JavaScript 监听输入；
- 输入时请求 `/genealogies/<id>/members/options?q=关键词`；
- 后端最多返回少量候选；
- 页面不加载全量成员。

汇报时可以说：

为了避免 10 万级数据下浏览器 OOM，系统采用动态搜索下拉，而不是一次性加载全部成员。

---

### 4. 树状族谱展示

相关文件：

```text
genealogy_app/services.py
genealogy_app/routes.py
templates/tree.html
templates/base.html
```

实现方式：

- `query_subtree()` 使用递归 CTE 查询根成员下方指定层数；
- 查询结果组装为树结构；
- `tree.html` 递归渲染节点；
- CSS 负责画出树状连接线；
- 夫妻通过横向并列展示。

汇报时可以说：

树状展示不是一次渲染整个族谱，而是选择根成员和最大层数，保证大数据下也能稳定展示。

---

### 5. 祖先查询

相关文件：

```text
genealogy_app/services.py
templates/ancestors.html
```

实现方式：

- `query_ancestors()` 使用 `WITH RECURSIVE`；
- 从当前成员向上查父母；
- 返回祖先姓名、性别、生卒年和深度。

注意：

当前页面是表格展示，不是树状图。这是后续可改进点。

---

### 6. 亲缘路径查询

相关文件：

```text
genealogy_app/services.py
templates/kinship.html
```

实现方式：

- 读取当前族谱成员；
- 读取亲子关系和婚姻关系；
- 构建无向图；
- 使用 BFS 从起点找终点；
- 找到后回溯路径，并标注关系类型。

汇报时可以说：

亲缘路径查询本质是图搜索问题，系统使用 BFS 找出两名成员之间的连通路径。

## 八、代码入口在哪里


程序入口是 `app.py`，它调用 `genealogy_app/__init__.py` 创建 Flask 应用。主要页面逻辑在 `genealogy_app/routes.py`，数据库模型在 `genealogy_app/models.py`，递归查询和亲缘路径查询在 `genealogy_app/services.py`。

---

## 九、最核心的代码是哪部分


最核心的是三部分：

第一，`models.py` 定义了数据库模型，对应数据库设计。

第二，`services.py` 实现了递归祖先查询、分支树查询和亲缘路径 BFS。

第三，`routes.py` 把页面操作和数据库操作连接起来，实现完整应用流程。

## 十、代码阅读建议

快速理解项目可以按这个顺序看：

1. 先看 `README.md`，理解项目功能。
2. 看 `数据库设计说明.md`，理解表结构。
3. 看 `genealogy_app/models.py`，对照数据库表。
4. 看 `genealogy_app/routes.py`，理解页面如何工作。
5. 看 `genealogy_app/services.py`，理解核心查询。
6. 看 `templates/`，理解页面展示。
7. 看 `交付物/sql/queries.sql` 和 `indexes.sql`，理解课程要求 SQL 和索引。
