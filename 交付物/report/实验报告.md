# 数据库大作业实验报告

## 一、课题名称

寻根溯源族谱管理系统

## 二、项目背景

族谱信息具有明显的层级关系、图结构关系和统计分析需求，适合作为数据库设计、递归查询、索引优化和应用层演示的综合案例。本项目实现一个基于 Flask 与 PostgreSQL 的族谱管理系统，支持成员维护、关系维护、递归祖先查询、树形预览和亲缘路径分析。

## 三、开发环境

- 操作系统：Windows
- 开发语言：Python 3
- Web 框架：Flask
- ORM：SQLAlchemy
- 数据库：PostgreSQL
- 演示数据库：SQLite

## 四、需求分析

系统需要完成以下核心功能：

1. 用户注册、登录、退出。
2. 用户只能管理自己创建或受邀参与的族谱。
3. 支持族谱创建、成员新增、编辑、删除。
4. 支持亲子关系和婚姻关系维护。
5. 支持成员姓名模糊查询。
6. 支持分支树形展示。
7. 支持祖先递归查询。
8. 支持两成员亲缘路径查询。
9. 支持批量生成与导入大规模测试数据。
10. 支持索引设计与性能分析。

## 五、数据库设计

### 5.1 实体与关系

本系统包含以下主要数据表：

- `users`
- `genealogies`
- `genealogy_collaborators`
- `members`
- `parent_child_relations`
- `marriages`

### 5.2 设计特点

- 通过 `genealogy_id` 隔离不同族谱的数据范围。
- 将亲子关系和婚姻关系拆分为独立关系表，避免在成员表中冗余存储可推导字段。
- 使用触发器校验跨表业务约束。
- 使用递归 CTE 支持祖先和后代层次查询。

### 5.3 规范化说明

本系统达到 3NF：

- 每张表只描述一个实体或一种关系。
- 非主属性完全依赖主键。
- 无明显传递依赖。

## 六、核心 SQL 设计

交付物中 `sql/queries.sql` 实现了以下 SQL：

1. 配偶与子女查询
2. 递归祖先查询
3. 平均寿命最长代际分析
4. 超过 50 岁且无配偶的男性成员查询
5. 早于本代平均出生年的成员查询
6. 四代后代查询

## 八、数据工程实现

### 8.1 数据生成

项目提供 `scripts/generate_data.py`，支持生成：

- 10 个族谱
- 100000 条成员数据
- 50000 条以上的单大族谱
- 多个 CSV 文件用于 PostgreSQL `COPY`

### 8.2 数据导入

导入顺序如下：

1. `users.csv`
2. `genealogies.csv`
3. `members.csv`
4. `parent_child_relations.csv`
5. `marriages.csv`
6. `sequence_sync.sql`

## 九、性能优化

主要通过以下方式优化：

- 为成员姓名模糊查询建立索引
- 为父节点、子节点建立索引
- 为婚姻关系的两侧成员建立索引
- 使用 `EXPLAIN ANALYZE` 对比索引前后查询计划
