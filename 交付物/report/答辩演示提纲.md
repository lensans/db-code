# 答辩演示提纲

## 一、演示顺序

1. 项目背景与目标
2. E-R 设计与数据表结构
3. GUI 功能演示
4. SQL 查询演示
5. 大规模数据生成与导入说明
6. 索引与性能分析

## 二、GUI 演示建议流程

1. 登录系统
2. 进入控制台
3. 创建一个演示族谱
4. 新增两个父母成员和一个子女成员
5. 建立亲子关系和婚姻关系
6. 演示成员模糊查询
7. 演示树形预览
8. 演示祖先查询
9. 演示亲缘路径查询

## 三、数据库演示建议

1. 展示 `schema.sql`
2. 展示 `queries.sql`
3. 展示 `indexes.sql`
4. 展示 `triggers.sql`
5. 说明 `COPY` 批量导入流程

## 四、重点回答方向

- 为什么拆分出亲子关系表和婚姻关系表
- 为什么使用递归 CTE
- 为什么使用触发器处理跨表校验
- 为什么需要 `pg_trgm` 索引
- 为什么 GUI 采用 Flask + Jinja2

## 五、答辩注意事项

1. GUI 演示建议使用中小规模数据，避免现场页面响应受大数据集影响。
2. SQL 性能测试建议提前准备截图，不要现场临时跑全部数据。
3. 若教师关注规范化，可重点说明 3NF 与冗余控制。
